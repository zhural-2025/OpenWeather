# OpenWeather Case

Telegram-бот для погоды на базе `pyTelegramBotAPI` и OpenWeather API.

## Структура проекта

- `bot.py`
- `weather_app.py`
- `storage.py`
- `requirements.txt`
- `.gitignore`
- `.env.example`
- `User_Data.json`
- `README.md`

## Настройка

1. Создайте и активируйте виртуальное окружение.
2. Установите зависимости:

```powershell
python -m pip install -r requirements.txt
```

3. Создайте `.env` на основе `.env.example` и заполните переменные:

```env
OW_API_KEY=your_openweather_key
BOT_TOKEN=your_telegram_token
```

## Запуск

```powershell
python bot.py
```

## Проверка за 5 минут

1. Откройте диалог с ботом и нажмите `/start`.
2. Проверьте `🌤 Текущая погода (город)`:
   - введите корректный город (например, `Moscow`);
   - введите невалидный город и убедитесь, что бот пишет `Город не найден.`.
3. Проверьте геолокацию:
   - нажмите `📍 Моя геолокация` и отправьте location;
   - нажмите `📅 Прогноз на 5 дней`, выберите день в inline-клавиатуре, затем `Назад`.
4. Проверьте `🏙 Сравнить города`:
   - отправьте два города через запятую (например, `Москва, Казань`);
   - убедитесь, что бот возвращает табличное сравнение температур.
5. Проверьте `🧪 Расширенные данные`:
   - отправьте город или координаты;
   - убедитесь, что в ответе есть блок качества воздуха на русском.
6. Проверьте `🔔 Уведомления`:
   - включите уведомления, задайте интервал в часах;
   - убедитесь, что в `User_Data.json` сохраняются `notifications.enabled` и `interval_h`.

## Полезно знать

- Inline-режим: в любом чате Telegram введите `@<username_бота> <город>`.
- Для inline-функции включите ее в BotFather через `/setinline`.
- API-кэш хранится в `./.cache/` и живет 10 минут.
